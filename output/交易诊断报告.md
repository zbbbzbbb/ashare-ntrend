## 一、入场选股全流程

每天收盘后，经过四层过滤决定买什么：

```
沪深300 MACD牛熊  ──→  行业轮动评分  ──→  N型突破选股  ──→  仓位+止损执行
    (第1层)             (第2层)           (第3层)           (第4层)
```

### 第1层：大盘择时

用沪深300指数的 MACD 判断市场状态。**三个条件必须同时满足**才允许开新仓：DIF > DEA、柱状图绝对值在扩大、DIF在上升。

2015-2025年间牛市占比仅 25.5%，一半以上的日子不能买入。

### 第2层：行业轮动

从440个申万三级行业中选出前5个。每个行业用成分股等权合成价格曲线，按 MACD看多(25%) + 相对强度(35%) + 成交量放大(20%) + N型形态(20%) 打分。

### 第3层：N型突破选股

```
价格
 │    ┌── 第一段上升（T-20 → T-10，涨幅 > 2%）
 │   ╱
 │  ╱  ╲
 │ ╱    ╲___ 回调（T-10 → T-5，跌幅 < -1%）
 │╱         ╲___ 今天收盘突破前期高点 → 买入信号
─┼────────────────────────────→ 时间
 T-20     T-10  T-5   今天
```

每只股票用 MACD看多(25%) + N型质量(30%) + 放量(20%) + BBI上方(15%) + N型检测(10%) 加权打分。硬性过滤：MACD必须看多 **且** 成交量必须放量。合格不足3只时放宽N型要求。

### 第4层：止损与止盈

| 入场位置 | 止损规则 |
|---|---|
| 入场价 **高于** BBI | 连续两天收盘 < BBI → 清仓（BBI追踪止损） |
| 入场价 **低于** BBI | 收盘跌破入场价 -3% → 清仓（固定百分比止损） |

止盈：盈利 ≥ +20% → 卖出 50%，剩余继续持有等趋势止损。

## 七、Bug修复记录

### Bug 1：BBI止损日期索引错误

**位置**：`src/portfolio/risk.py` `update()` 方法

**问题**：`iloc[idx]` 传入 `idx=-1`，永远读取BBI DataFrame的最后一行（2025年的数据），而非当前交易日。导致 `above_bbi=True` 的买入永远不会触发BBI止损。

**修复**：
- `src/backtest/engine.py`：在每个交易日计算每只持仓股票的BBI DataFrame中对应日期的正确行号，传入 `bbi_idx_map`
- `src/portfolio/execution.py`：`process_daily()` 接收 `bbi_idx_map` 替代 `bbi_idx: int`，分发给 `risk_manager.update()` 和 `_execute_buy()`
- `src/portfolio/risk.py`：`update()` 使用正确的行号

### Bug 2：新买入时BBI数据缺失

**位置**：`src/backtest/engine.py`

**问题**：`bbi_data` 只包含已持仓股票的BBI数据，新买入的候选股 `bbi_df=None`，导致 `_execute_buy()` 中 `above_bbi` 永远为 `False`，所有交易都走情况A（-3%止损）。

**修复**：在构建 `bbi_idx_map` 时，同时将候选股的BBI数据填入 `bbi_data`。
