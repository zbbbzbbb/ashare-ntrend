## 0. 角色与目标

你是一名资深量化研究员兼工程师。

**核心目标**：在 A股，构建并严格验证一套**纯多头、自顶向下**的策略，
追求**稳健的样本外长期复合收益**，同时把回撤控制在可接受范围。

收益应主要来自：
- 正确的**市场状态（牛/熊/震荡）判断**——决定何时承担风险、承担多少；
- **领涨板块/行业的轮动**——把资金集中到正在被增量资金推动的方向；
- **趋势的参与与延续**——在上升结构中进场、在结构破坏时退出。

收益**不应**主要依赖：财务因子、统计套利、均值回归套利、黑箱机器学习。

资本保全应主要靠**择时、趋势退出、仓位管理**，而不是过度分散。

> ⚠️ 注意：以上是**待验证的投资假设**，不是已知结论。你的首要任务之一就是用数据
> 检验"自顶向下三层框架是否真的比朴素基准更好"。如果某一层不产生增量价值，必须如实指出。

## 1. 你要研究的框架（结构固定，方法开放）

整体结构是固定的、三层、自顶向下：

```
第一层  大盘择时        →   第二层  行业/板块轮动   →   第三层  个股/标的选择
(决定敢不敢上仓、上多少)      (决定钱往哪个方向集中)        (决定具体买什么、怎么进出)
```

**但每一层"怎么做"都是开放研究问题**，你必须自己探索并用证据择优：

### 第一层 · 大盘择时（如何"给市场打分"）
开放问题：
- 用哪个/哪些指数代表"大盘"？（宽基？大盘 vs 中小盘？全市场？多指数综合？）
- 用什么方法判断状态？（趋势类、动量类、波动类、广度/市场宽度、资金面如北向/两融、
  估值分位、宏观利率……）——**自由选择与组合，并比较**。
- 状态是二元（上/不上）还是连续（仓位刻度）？
- 不同参数/不同指数下，状态判断的**稳健性**如何？

### 第二层 · 行业/板块轮动（如何"找方向"）
开放问题：
- 用什么行业体系？（申万一/二/三级、中证行业、主题板块……）
  ⚠️ **数据现实**：申万行业指数(801xxx/850xxx)在本数据湖**没有日线**，
  只能用成分股自行合成，或改用**有真实日线**的中证行业/主题指数。请先核实数据再设计。
- 用什么衡量"领涨/有增量资金流入"？（相对强度、动量、量能、北向/两融净流入、
  换手、资金集中度……）
- 选几个行业？集中还是分散？轮动频率多高？
- **关键**：行业层到底贡献多少增量价值？（见第4节"分层归因"——这是必答题）

### 第三层 · 个股/标的选择（如何"挑具体标的"）
开放问题：
- 在选中的方向里，用什么挑标的？（趋势结构、突破、相对强度、量价、流动性、
  资金面、龙头属性……）——**自由设计，并与多种排序法对比**。
- 排序打分如何构造，才能**真正区分好坏**（而不是大量并列）？
- **任何入场信号，先用事件研究测它的"超额收益时间尺度"**：超额集中在未来几天？
  几天后衰减？这直接决定持有期与退出规则——**先量出edge的半衰期，再设计退出**。

### 表达载体（开放！不要默认只能买个股）
允许的持有标的：**A股个股、上市ETF（含宽基/行业/主题）、现金/货币或债券ETF**。
（不允许：做空、杠杆、融资、期货、期权、杠杆ETF。）

> 这意味着：每一层的"信号"和"用什么工具去表达这个信号"是两件事。
> 例如大盘择时的多头观点，可以用个股表达，也可以用宽基ETF表达——**哪个更好是实证问题**。
> 不要先入为主地认为"必须靠选个股赚钱"。

## 2. 硬性约束

- **纯多头**；无做空/杠杆/融资/期货/期权/杠杆ETF。
- **资金规模 1,500,000 RMB**；所有策略必须在该规模下可执行（尊重流动性/参与率上限）。
- **可用数据**（仅限）：复权价、成交量/额、换手、波动、市值、PE/PB、股息率、
  ETF数据、指数数据、市场宽度、北向持股、两融、宏观（利率等）。
- **回撤偏好**：最大回撤 ≤ 35%（更优 ≤ 30%）；不接受需要长期 >50% 回撤的方案。

## 4. 方法论铁律（这是与普通回测的根本区别，必须全部满足）

1. **分层边际归因（必答）**：必须分别测量每一层的**增量价值**——
   逐层加入/移除，看样本外指标如何变化。明确回答："择时层值多少？行业层在择时之上
   再加多少？个股层在前两层之上再加多少？" 任何一层若无增量价值，必须如实写出。

2. **必须跑赢朴素基准，否则复杂度不成立**。至少对比：
   (a) 标的买入持有；(b) 单一宽基指数 + 你的择时层（最朴素的自顶向下表达）。
   如果完整三层策略打不赢 (b)，说明后两层是负担而非价值。

3. **成本必须真的扣到净值里**（不是写在文档里）：佣金、印花税（卖出）、
   **滑点、冲击成本**，全部计入。并跑 **正常成本** 与 **2×成本压力测试**。

4. **无前视 / PIT 纪律**：信号只能用截至当日收盘的信息；执行用次日或当日收盘
   并讲清楚约定。**主动检查 look-ahead**（一个常见陷阱：把当日信号用在当日收益上）。

5. **edge 的时间尺度先用事件研究量化**：在做任何完整回测前，先用事件研究测入场信号
   的未来 5/10/20/40 日超额收益（对比同期可投基准），看 edge 在哪、何时衰减、
   中位数是否为正。**退出规则必须匹配 edge 的真实半衰期。**

6. **严格防过拟合**：Walk-Forward、CPCV、Purged CV + Embargo、参数稳定性、
   **试验次数追踪 + Deflated Sharpe**、分状态(regime)拆解、**最终 holdout 只看一次**。

7. **闲置资金计息**：若策略经常空仓，空仓期应按货币市场利率(可用 shibor)计息，
   否则会系统性低估收益与 Sharpe。

8. **风险指标要完整**：CAGR、年化波动、Sharpe（注明是否减无风险）、Sortino、
   最大回撤、Calmar、盈亏比、胜率、平均持仓、换手、暴露占比。

---

## 5. 成功指标

主：**样本外 CAGR** 与 **样本外 Calmar**。
次：终值、Sharpe/Sortino、最大回撤、盈亏比、跨状态稳定性。
偏好：在回撤约束内、用**最简单**的结构拿到最稳健的样本外复合收益。
（同等表现下，越简单、越低换手、对成本越不敏感者越优。）

---

## 6. 过程与交付物

建议流程：
1. 数据审计 → 2. 基础设施（快速、向量化、可并行的回测框架） →
3. 第一层择时研究 → 4. 第二层行业研究 → 5. 第三层个股研究 →
6. 组合构建（仓位/风控/交易约束作为**研究产出**，不预设） →
7. 防过拟合验证 → 8. 最终 holdout → 9. 最终决策。

每个阶段都要记录：发现、假设、风险、下一步。

**工程要求**：独立工作单元（多因子/多标的/多参数/多配置）默认**并行 + 向量化**，
用满多核；不要用单核 for 循环跑可向量化的计算。

---

## 7. 最终决策原则

选择满足以下条件的方案：样本外 CAGR 稳健、状态间表现一致、回撤可接受、
**且其复杂度被分层归因证明是值得的**。

拒绝：过拟合、对成本脆弱、打不赢朴素基准、无法解释收益来源的方案。

最终策略应当**适合用真实资金长期部署**。

## 附录 A（可选 · 可整段删除）：来自一次"独立前序研究"的经验性先验

> 下面是**别处一次独立研究**得出的若干观察。**把它们当作待你独立证实或证伪的假设，
> 不要直接采信**。若你想要完全空白的起点，请删除本附录。

- 某些写法里，**"大盘择时"层往往是价值最大的一层**；而单纯靠价格形态选个股，
  其超额收益可能只是"几天的短脉冲、之后衰减、中位数为负"。建议**优先用事件研究
  量化每个信号的edge时间尺度**，再决定持有期。
- **择时的表达载体很关键**：同一个择时观点，用不同宽基指数表达，风险收益差别很大；
  "宽基ETF + 择时"这条朴素基准常常很难被复杂选股打败——所以(第4节)把它设成必过基准。
- A股**个股层面的"追动量/追趋势"未必有正edge**（截面上可能反转占优）；
  这意味着第三层的信号设计需要实证，不能照搬美股动量直觉。
- **成本会吃掉短持有期策略的大部分薄edge**；交易越频繁越要把滑点/冲击算实。
- **数据现实**：申万行业指数无日线，需合成或改用有真实日线的指数（务必先核实）。
